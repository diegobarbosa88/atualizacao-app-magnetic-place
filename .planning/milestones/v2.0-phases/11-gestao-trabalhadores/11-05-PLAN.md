## Plano 11-05: Histórico de Períodos de Emprego

### Objetivo
Quando um trabalhador sai e volta à empresa, o período anterior é preservado em histórico e um novo período começa — sem perder dados.

### Tarefas

#### Tarefa 1: Criar tabela SQL no Supabase
**Action:**
Executar no SQL Editor do Supabase:

```sql
CREATE TABLE worker_employment_history (
  id uuid DEFAULT gen_random_uuid() PRIMARY KEY,
  worker_id text NOT NULL,
  data_inicio text NOT NULL,
  data_fim text,
  created_at timestamptz DEFAULT now()
);
ALTER TABLE worker_employment_history ENABLE ROW LEVEL SECURITY;
CREATE POLICY "Allow all on worker_employment_history" ON worker_employment_history FOR ALL USING (true) WITH CHECK (true);
```

**Acceptance criteria:**
- Tabela `worker_employment_history` existe no Supabase
- RLS ativo com policy "Allow all"
- Colunas: id (uuid), worker_id (text), data_inicio (text), data_fim (text), created_at (timestamptz)

#### Tarefa 2: Salvar período ao guardar worker com dataFim
**Action:**
Modificar `handleSaveWorker` em `TeamContext.jsx`:
1. Quando `workerForm.dataFim` é preenchido E o worker já tem período ativo no histórico sem `data_fim`, guardar esse período com `data_fim = dia anterior`
2. Quando `workerForm.dataInicio` é preenchido E não existe período ativo sem `data_fim`, criar novo registo em `worker_employment_history`

```javascript
// No início de handleSaveWorker:
const saveEmploymentHistory = async (workerId, dataInicio, dataFim) => {
  if (!dataInicio) return;
  
  // Buscar último período sem data_fim
  const { data: lastPeriod } = await supabase
    .from('worker_employment_history')
    .select('*')
    .eq('worker_id', workerId)
    .is('data_fim', null)
    .single();
  
  if (lastPeriod && dataFim) {
    // Fechar período anterior (dia antes)
    const prevDate = new Date(dataFim);
    prevDate.setDate(prevDate.getDate() - 1);
    const prevDateStr = prevDate.toISOString().split('T')[0];
    await saveToDb('worker_employment_history', lastPeriod.id, {
      ...lastPeriod,
      data_fim: prevDateStr
    });
  }
  
  if (!lastPeriod || dataFim) {
    // Criar novo período
    await saveToDb('worker_employment_history', crypto.randomUUID(), {
      worker_id: workerId,
      data_inicio: dataInicio,
      data_fim: dataFim || null
    });
  }
};
```

Chamar `saveEmploymentHistory(id, workerForm.dataInicio, workerForm.dataFim)` antes de `saveToDb('workers', ...)`.

**Acceptance criteria:**
- `grep -n "worker_employment_history" src/features/admin/contexts/TeamContext.jsx` retorna pelo menos 3 matches
- Código trata os 3 casos: fechar período anterior, criar novo período, worker novo sem histórico
