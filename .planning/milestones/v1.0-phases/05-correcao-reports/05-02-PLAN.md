   ```javascript

   ```
   If target_type is wrong, notification won't reach the specific client.

2. **Missing target_client_id** — Admin notification must include `target_client_id: correcao.client_id` so client's filter (line 110) can match it.

3. **Client notification filter issue** (line 108-114):
   ```javascript
   const matchTarget = n.target_type === 'client';
   const matchClientId = String(n.target_client_id) === String(initialClientId);
   ```
   If admin sends with target_type='specific' but client's filter expects 'client', notification won't appear.

4. **handleAcceptContestation not triggered** — Client portal may receive notification but not display the accept/reject buttons if `notif.payload?.type !== 'counter_proposal'`.

**Fix required:**
- Ensure admin's counter-proposal has `target_type: 'client'` AND `target_client_id: correcao.client_id`
- Or adjust client's filter to accept 'specific' target_type with matching client_id

Check if the notification from app.jsx line 2906 has correct targeting.
