# Phase 5: Correção do Sistema de Reports - PLANO 05-03

## Problema
O código da `CorrecoesAdmin` é confuso com lógicas misturadas entre quick e precision reports.

## Regras de negócio (confirmadas)
- **Quick Report** ("Rápido"): Admin pode editar horas, aceitar e aplicar, rejeitar, enviar contra-proposta
- **Precision Report** ("Precisão"): Admin pode editar horas de qualquer dia, aceitar e aplicar, rejeitar, enviar contra-proposta
- Ambos permitem admin editar horas — a diferença é apenas a origem dos dados

## Solução: Separar routing no início do render

### Ficheiro a modificar
`src/app.jsx` — linhas 2166-3073 (CorrecoesAdmin render)

### Implementação

#### Passo 1: Separar notifications no início do map (linha ~2166)

```javascript
correctionNotifications.map(notif => {
  const isQuickReport = notif.payload?.reportType === 'quick';
  const isPrecisionReport = notif.payload?.reportType === 'precision';

  if (isQuickReport) {
    return <QuickReportCorrectionCard key={notif.id} notif={notif} {...sharedProps} />;
  }
  if (isPrecisionReport) {
    return <PrecisionReportCorrectionCard key={notif.id} notif={notif} {...sharedProps} />;
  }
  // Fallback: legacy handling for notifications without reportType
  return <LegacyCorrectionCard key={notif.id} notif={notif} {...sharedProps} />;
});
```
