<objective>
Implement D-01: Separate state structures for Quick and Precision reports. Currently the monolithic CorrecoesAdmin uses a single `editingDrafts` state that can cause cross-contamination when editing multiple notifications of different types.

Purpose: Complete state isolation prevents editing one report from affecting another
Output: Two separate state hierarchies for quick and precision corrections
</objective>
