## Plano 11-06: Histórico de Atribuição de Horários

### Objetivo
Quando um worker para de ser atribuído a um horário (dataFim), o período é guardado em histórico. Reatribuir o mesmo horário cria um novo período — o histórico anterior é preservado.

### Tarefas

#### Tarefa 1: Criar tabela SQL no Supabase
**Action:**
Executar no SQL Editor do Supabase:

```sql
CREATE TABLE worker_schedule_history (
  id uuid DEFAULT gen_random_uuid() PRIMARY KEY,
  worker_id text NOT NULL,
  schedule_id text NOT NULL,
  data_inicio text NOT NULL,
  data_fim text,
  created_at timestamptz DEFAULT now()
);
ALTER TABLE worker_schedule_history ENABLE ROW LEVEL SECURITY;
CREATE POLICY "Allow all on worker_schedule_history" ON worker_schedule_history FOR ALL USING (true) WITH CHECK (true);
```

**Acceptance criteria:**
- Tabela `worker_schedule_history` existe no Supabase
- RLS ativo com policy "Allow all"
- Colunas: id (uuid), worker_id (text), schedule_id (text), data_inicio (text), data_fim (text), created_at (timestamptz)

#### Tarefa 2: Modificar ScheduleContext para guardar em histórico
**Action:**
Em `ScheduleContext.jsx`, modificar `handleUnassignSchedule` para guardar em `worker_schedule_history`:

```javascript
// Modificar handleUnassignSchedule:
const handleUnassignSchedule = useCallback(async (workerId, scheduleId) => {
  const supabase = window.supabaseInstance;
  const w = workers.find(worker => worker.id === workerId);
  if (!w) return;
  
  const scheduleDates = w.assignedScheduleDates || {};
  const currentDates = scheduleDates[scheduleId] || {};
  
  if (currentDates.dataInicio) {
    // Guardar período em histórico
    await saveToDb('worker_schedule_history', crypto.randomUUID(), {
      worker_id: workerId,
      schedule_id: scheduleId,
      data_inicio: currentDates.dataInicio,
      data_fim: new Date().toISOString().split('T')[0]
    });
  }
  
  // Remover do assignedScheduleDates (não apaga do histórico)
  delete scheduleDates[scheduleId];
  
  await saveToDb('workers', w.id, { 
    ...w, 
    assignedScheduleDates: scheduleDates 
  });
}, [workers, saveToDb]);
```

Também modificar `handleAssignScheduleWithDates` para criar novo período em histórico (em vez de substituir):

```javascript
// Modificar handleAssignScheduleWithDates:
const handleAssignScheduleWithDates = useCallback(async (workerId, scheduleId, dataInicio, dataFim) => {
  const w = workers.find(worker => worker.id === workerId);
  if (!w) return;
  
  // Criar novo período em histórico
  await saveToDb('worker_schedule_history', crypto.randomUUID(), {
    worker_id: workerId,
    schedule_id: scheduleId,
    data_inicio: dataInicio,
    data_fim: dataFim || null
  });
  
  // Atualizar assignedScheduleDates com último período
  const scheduleDates = w.assignedScheduleDates || {};
  scheduleDates[scheduleId] = { dataInicio, dataFim };
  
  const currentAssigned = w.assignedSchedules || [];
  const nextAssigned = currentAssigned.includes(scheduleId) 
    ? currentAssigned 
    : [...currentAssigned, scheduleId];
  
  await saveToDb('workers', w.id, { 
    ...w, 
    assignedSchedules: nextAssigned,
    assignedScheduleDates: scheduleDates 
  });
}, [workers, saveToDb]);
```

**Acceptance criteria:**
- `grep -n "worker_schedule_history" src/features/admin/contexts/ScheduleContext.jsx` retorna pelo menos 4 matches
- `handleUnassignSchedule` guarda em histórico
- `handleAssignScheduleWithDates` cria novo período em histórico

#### Tarefa 3: Adicionar botão 📅 no ScheduleManager para ver histórico de atribuições
**Action:**
Em `ScheduleManager.jsx`, junto ao worker que tem um horário atribuído, adicionar botão que mostra o histórico:

1. Adicionar estado para histórico:
```jsx
const [showScheduleHistory, setShowScheduleHistory] = useState({ show: false, workerId: null, workerName: '', scheduleId: null, scheduleName: '', history: [] });
```

2. Função para carregar histórico:
```jsx
const loadScheduleHistory = async (workerId, workerName, scheduleId, scheduleName) => {
  const { data } = await supabase
    .from('worker_schedule_history')
    .select('*')
    .eq('worker_id', workerId)
    .eq('schedule_id', scheduleId)
    .order('created_at', { ascending: false });
  setShowScheduleHistory({ show: true, workerId, workerName, scheduleId, scheduleName, history: data || [] });
};
```

3. Botão junto ao horário atribuído (onde aparece o horário do worker):
```jsx
<button 
  onClick={() => loadScheduleHistory(w.id, w.name, s.id, s.name)}
  className="ml-1 text-xs text-slate-400 hover:text-indigo-600"
  title="Histórico de atribuição"
>📅</button>
```

4. Modal que mostra a lista de períodos:
```jsx
{showScheduleHistory.show && (
  <div className="fixed inset-0 bg-black/50 flex items-center justify-center z-50" onClick={() => setShowScheduleHistory({ show: false, workerId: null, workerName: '', scheduleId: null, scheduleName: '', history: [] })}>
    <div className="bg-white p-6 rounded-2xl max-w-md w-full mx-4 shadow-2xl" onClick={e => e.stopPropagation()}>
      <h3 className="text-lg font-black text-indigo-700 mb-1">Histórico: {showScheduleHistory.scheduleName}</h3>
      <p className="text-sm font-bold text-slate-500 mb-4">{showScheduleHistory.workerName}</p>
      {showScheduleHistory.history.length === 0 ? (
        <p className="text-sm text-slate-400 text-center py-4">Sem períodos registados</p>
      ) : (
        <div className="space-y-3 max-h-64 overflow-y-auto">
          {showScheduleHistory.history.map((h, i) => (
            <div key={h.id} className="flex justify-between items-center p-3 bg-slate-50 rounded-xl">
              <span className="text-sm font-bold text-slate-600">{h.data_inicio}</span>
              <span className="text-slate-400">→</span>
              <span className="text-sm font-bold text-indigo-600">{h.data_fim || 'Atual'}</span>
            </div>
          ))}
        </div>
      )}
    </div>
  </div>
)}
```

**Acceptance criteria:**
- Botão 📅 visível junto ao horário na lista de workers
- Modal abre e mostra todos os períodos históricos de atribuição
