<task type="auto">
  <name>Task 2: Create useBlockTemplate hook</name>
  <files>src/hooks/useBlockTemplate.js</files>
  <action>
Create `useBlockTemplate.js` hook for CRUD operations on template blocks.

**Hook signature:**
```javascript
export function useBlockTemplate(supabase, { onError } = {}) {
  // Returns: { templates, loading, saving, createTemplate, updateTemplate, deleteTemplate, getTemplate }
}
```

**Data model (Supabase tables):**

Existing `document_templates` table (check schema):
- id, name, description, html_content, created_at, updated_at

New approach: Store JSON blocks in `template_blocks` table:
```javascript
{
  id: uuid,
  template_id: uuid (FK to document_templates.id),
  type: 'title' | 'subtitle' | 'paragraph' | 'signature',
  content: text,
  order: integer,
  created_at, updated_at
}
```

**Functions:**

1. `createTemplate(name, description, blocks)` — creates document_templates row, then bulk inserts template_blocks rows
2. `updateTemplate(templateId, name, description, blocks)` — updates document_templates, deletes existing template_blocks, bulk inserts new blocks
3. `deleteTemplate(templateId)` — deletes template_blocks first, then document_templates
4. `getTemplates()` — fetches document_templates, then fetches corresponding template_blocks for each, returns merged structure with blocks array
5. `getTemplate(templateId)` — single template with blocks

**Blocks format (internal):**
```javascript
[{ type: 'title', content: 'My Document Title', order: 0 }]
```

**Storage format (Supabase):**
- `template_blocks.template_id` references `document_templates.id`
- Order preserved via `order` field

**Error handling:**
- Wrap all operations in try/catch
- Call `onError?.(err)` on failure
- console.error with descriptive message

**Loading states:**
- `loading` — true during getTemplates/getTemplate
- `saving` — true during createTemplate/updateTemplate/deleteTemplate
