# Plano: Fase 7.2 - Modularização Admin (Horários e Despesas)

Este plano foca na extração das funcionalidades de gestão de horários e despesas para componentes dedicados.

## 🎯 Objetivos
- Criar componente independente para a aba "Horários".
- Criar componente independente para a aba "Despesas".
- Remover os estados `scheduleForm` e `expenseForm` do `app.jsx`.

## 🛠️ Tarefas

### 1. Extrair Gestão de Horários
- [ ] Criar `src/features/admin/ScheduleManager.jsx`.
- [ ] Mover a lógica de `scheduleForm` e a renderização da gestão de turnos.
- [ ] Implementar a lógica de associação de trabalhadores a turnos.
- [ ] Consumir `schedules`, `workers` e `saveToDb`/`handleDelete` via `useApp()`.

### 2. Extrair Gestão de Despesas
- [ ] Criar `src/features/admin/ExpenseManager.jsx`.
- [ ] Mover a lógica de `expenseForm` e a renderização da tabela de despesas.
- [ ] Implementar filtros de despesas se houver (ou manter a lógica atual).
- [ ] Consumir `expenses` e `saveToDb`/`handleDelete` via `useApp()`.

### 3. Integração no AdminDashboard
- [ ] Importar e utilizar `ScheduleManager` e `ExpenseManager` no `AdminDashboard`.
- [ ] Limpar estados e handlers redundantes no `app.jsx`.

## 🧪 Verificação
- [ ] Criar um novo horário e verificar se os trabalhadores associados são atualizados.
- [ ] Registar uma despesa e validar o cálculo no Dashboard Geral.

## ⚠️ Riscos
- **Associações Complexas**: A relação entre horários e trabalhadores deve ser mantida cuidadosamente durante a transição.
